# 🌲 樹與圖形理論複習筆記 (Trees & Graph Theory)

## 壹、樹與二元樹 (Trees & Binary Trees)

### 1. 表達式樹 (Expression Tree) 與遍歷
表達式樹是一種特殊的二元樹，其特性如下：
- **葉子節點 (Leaf Nodes)**: 儲存「運算元」(Operands)，如數字。
- **內部節點 (Internal Nodes)**: 儲存「運算子」(Operators)，如 `+`, `-`, `*`, `/`。

#### 🔍 Quiz IV 實戰解析 (算式求值)


**中序遍歷 (Inorder Traversal)** 順序：`左子樹` $\rightarrow$ `根節點` $\rightarrow$ `右子樹`。

* **完整算式**: $((3 \times 4) + (10 \div 2)) - 3$
* **運算步驟**:
    1.  先算括號內：$3 \times 4 = 12$
    2.  再算除法：$10 \div 2 = 5$
    3.  執行加法：$12 + 5 = 17$
    4.  最後減法：$17 - 3 = 14$
* **計算結果**: **14**

### 2. 樹的分類 (Classification)

| 分類邏輯 | 類型 | 定義與特性 |
| :--- | :--- | :--- |
| **形狀限制型** | **完全二元樹 (Complete)** | 除了最後一層外，每一層皆填滿，且節點由左至右排列。 |
| **規則準則型** | **二元搜尋樹 (BST)** | 左子樹值 $<$ 根節點 $<$ 右子樹值。 |
| **規則準則型** | **堆積 (Heap)** | 滿足父節點必定大於 (Max) 或小於 (Min) 子節點的規則。 |

## 貳、圖形理論 (Graph Theory)

### 1. 基本定義 $G(V, E)$
- **節點 (Vertex, V)**: 代表實體（如地標、伺服器 IP）。
- **邊 (Edge, E)**: 連接節點的線（代表路徑、連線）。
- **權重 (Weight)**: 代表距離、成本或延遲時間。

### 2. 重要分類

| 類型 | 描述 | 生活實例 |
| :--- | :--- | :--- |
| **無向圖 (Undirected)** | 邊沒有方向性。 | 雙向道路、Facebook 好友。 |
| **有向圖 (Directed)** | 邊具備特定方向。 | 單行道、Instagram 追蹤關係。 |
| **連通圖 (Connected)** | 任意兩個節點都有路徑相通。 | 完整通車的公路網。 |
| **無環圖 (Acyclic)** | 圖中不存在任何迴圈路徑。 | 家族譜系樹、編譯依賴圖。 |

### 3. 圖的遍歷：廣度優先搜尋 (BFS)


- **核心機制**: 從起點開始，**逐層 (Layer by Layer)** 探索。
- **資料結構**: 使用 **佇列 (Queue)** 實作（先進先出 FIFO）。

#### 🔍 Quiz IV 實戰解析 (遍歷順序)
假設從節點 `0` 開始走訪：
1. **層級 0**: `0`
2. **層級 1**: `1, 2, 3` (與 0 直接相連)
3. **層級 2**: `4, 6, 7` (分別與 1, 2, 3 相連)
4. **層級 3**: `5` (與 4 相連)
* **最終順序**: `0 -> 1 -> 2 -> 3 -> 4 -> 6 -> 7 -> 5`

## 💡 參、總結對比：樹 vs. 圖

| 特性 | 樹 (Tree) | 圖 (Graph) |
| :--- | :--- | :--- |
| **結構** | 階層式 (Hierarchical) | 網狀式 (Network) |
| **環 (Cycle)** | **絕對不允許**有環 (Acyclic) | 可以有環 (Cyclic) |
| **父節點** | 每個節點最多一個父節點 | 節點可以有多個連接源頭 |
| **路徑** | 任意兩節點間路徑唯一 | 兩節點間可能有多條路徑 |
