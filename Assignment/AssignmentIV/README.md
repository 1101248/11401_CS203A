## 1. 實作方法說明

### 整數部分：Mid-Square Method
這次選擇用「中平方法」。邏輯是把 Key 平方之後，取中間的位數來當索引。
* **原因**：如果只用 Division (Key % m)，當遇到像 21, 22, 23 這種連號的數字時，索引會排在一起。中平方法平方後可以把數字拉開，減少碰撞。
* **做法**：`square = key * key` -> 轉字串 -> 去掉頭尾取中間 -> `% table_size`。

### 字串部分：ASCII Sum
簡單把字串每個字元的 ASCII 加起來。
* **原因**：雖然這方法比較基礎，但至少比只看第一個字元好，可以避免 "cat" 和 "car" 直接撞在一起。
